< how it works

This article will lead you through most llamechat endpoints and give you a basic understanding of how a llamechat client/server interacts + what endpoints you need to reach to complete certain operations.

creating/deleting accounts authentication creating/deleting channels sending messages

creating/deleting accounts

To create an account, populate the username and password headers of a POST request to /accounts/create/. This endpoint will succeed when complete.

{
	"success": {
		"code": 200,
		"message": "Account created."
	}
}

To delete an account, populate the username and password headers of a POST request to /accounts/delete/. This endpoint will succeed when complete

{
	"success": {
		"code": 200,
		"message": "Account nuked."
	}
}

authentication

Tokens are currently not planned as a method of authentication.

The /accounts/authenticate/ endpoint is used as a comprimise for tokens, send a POST request with specified username and password headers to authenticate that account, the endpoint will succeed if the authentication is successful.

{
	"success": {
		"code": 200,
		"message": "Valid."
	}
}

creating/deleting channels

To create a channel, populate the username, password, and name headers of a POST request to /channels/create/. This endpoint will succeed when complete.

{
	"success": {
		"code": 200,
		"message": "Channel created."
	}
}

To delete a channel, populate the username, password, and name headers of a POST request to /channels/delete/. This endpoint will succeed when complete.

{
	"success": {
		"code": 200,
		"message": "Channel nuked."
	}
}

sending messages

To send a message, populate the username, password, and message headers of a POST request to /channels/any/send/. (where any is the name of the channel) This endpoint will return a success when the action is complete.

{
	"success": {
		"code": 200,
		"message": "Message sent."
	}
}
